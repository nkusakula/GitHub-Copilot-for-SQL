# GitHub Copilot — Database Instructions & Custom Instructions in SSMS
### Customer Demo Script | AdventureWorks Database

---

## Setup Checklist (Before the Demo)

1. **SSMS 22.3+** installed with the AI Assistance workload
2. Signed into GitHub account with Copilot access
3. Connected to the `adventure_works` database
4. Copilot Chat window open: **View → GitHub Copilot Chat**
5. `src/database-instructions/` scripts ready to execute
6. `.github/copilot-instructions.md` copied to `%USERPROFILE%\copilot-instructions.md`

---

## Overview

| Feature | Where It Lives | Who Sets It Up | Who Benefits |
|---|---|---|---|
| **Database instructions** | Inside the database (extended properties) | DBA / data owner | Every user of that database |
| **Database constitution** | Inside the database (extended properties) | DBA / data owner | Every user of that database |
| **Custom instructions** | Markdown file on disk | Individual developer | That developer, in all sessions |

> *"Database instructions teach Copilot about **your data**. Custom instructions teach Copilot about **you**."*

## Part 1 — Database Instructions

### Demo 1 — Expose the Problem: Ambiguous Business Logic

**Step 1:** Start a new thread (`Ctrl+N`) and ask:

```
What is the total revenue generated by each product category in AdventureWorks?
```

**What to observe:** Copilot gives a plausible answer but has no idea that "revenue" means shipped orders only (`Status = 5`).

**What to say:**
> *"Technically correct — but business-incorrect. Without instructions, every developer might define revenue differently."*

---

### Demo 2 — Add a Table-Level Database Instruction

**Step 1:** In chat, type:

```
Add a database instruction for the SalesLT.SalesOrderDetail table to state that
revenue is defined as SUM(LineTotal) only for orders where the parent
SalesOrderHeader.Status = 5 (shipped). Cancelled or in-progress orders
should be excluded from revenue calculations.
```

**Step 2:** Copilot generates the `sp_addextendedproperty` T-SQL. Copy it, paste into a new query window, execute it.

> Alternatively, open and run `src/database-instructions/setup-database-instructions.sql` to install all instructions at once.

**What to say:**
> *"Copilot can't run this itself — it generates the script and I execute it. Copilot operates read-only in Ask mode."*

### Demo 6 — Database Constitution

**Step 1:** In chat, type:

```
Create a database constitution for adventure_works that states: all queries must
```

### Managing Instructions (Quick Reference)

```sql
-- Update an existing instruction
EXECUTE sp_updateextendedproperty
    @name       = N'AGENTS.md',
    @value      = N'Updated instruction text here.',
    @level0type = N'SCHEMA', @level0name = N'SalesLT',
    @level1type = N'TABLE',  @level1name = N'SalesOrderDetail';

-- Remove an instruction
EXECUTE sp_dropextendedproperty
    @name       = N'AGENTS.md',
    @level0type = N'SCHEMA', @level0name = N'SalesLT',
    @level1type = N'TABLE',  @level1name = N'SalesOrderDetail';

-- View all instructions
SELECT
    OBJECT_SCHEMA_NAME(ep.major_id)  AS SchemaName,
    OBJECT_NAME(ep.major_id)         AS TableName,
    COL_NAME(ep.major_id, ep.minor_id) AS ColumnName,
    CAST(ep.value AS NVARCHAR(MAX))  AS Instruction
FROM sys.extended_properties ep
WHERE ep.name IN (N'AGENTS.md', N'CONSTITUTION.md')
ORDER BY SchemaName, TableName, ColumnName;
```

## Part 2 — Custom Instructions

### Demo 7 — Show the Custom Instructions File

**Step 1:** In SSMS: **File → Open → File...** → navigate to `.github\copilot-instructions.md` in this repo.

**Walk through the key sections:**
- **Safety guardrails** — production-safe defaults, no DROP without WHERE
- **Code style** — aliases, schema-qualification, UPPERCASE keywords, no SELECT *
- **Business rules** — revenue definition, Status encoding baked in
- **Naming conventions** — `usp_`, `v_`, `fn_` patterns
- **Documentation** — header comment block template

**What to say:**
> *"This file lives in the `.github` folder — it's version-controlled and shared with the team. Each developer copies it to their `%USERPROFILE%` folder and SSMS applies it silently to every Copilot response."*

**Step 2:** Confirm it's enabled: **Tools → Options → GitHub → Copilot → Copilot Chat → Enable custom instructions**.

---

### Demo 8 — See Custom Instructions in Action

**Step 1:** Start a new thread (`Ctrl+N`) and ask:

```
Write a query to show the top customers by revenue with their order count
and average order value.
```

**What to observe in the response:**
- Table aliases match conventions (`c`, `soh`)
- Keywords uppercased (`SELECT`, `FROM`, `INNER JOIN`)
- Schema-qualified (`SalesLT.Customer`)
- No `SELECT *`
- `TOP` or filter applied automatically (safety guardrail)
- `Status = 5` filter applied (business rule)

**Step 2:** Check the **References** section in Copilot's response — `copilot-instructions.md` appears, confirming the file was applied.

**What to say:**
> *"I didn't ask for any of that. It came from the instructions file automatically."*

---

### Demo 9 — Stored Procedure with Header Documentation

**Step 1:** Ask:

```
Create a stored procedure that returns all orders for a given customer,
including product details, within a specified date range.
```

**What to observe:** Copilot generates `SalesLT.usp_GetCustomerOrders` with a full header comment block (Author, Date, Description, Parameters, Change History) and follows all naming and style conventions. Compare with `src/stored-procedures/usp_GetCustomerOrders.sql`.

## Comparison Summary

| | Database Instructions | Custom Instructions |
|---|---|---|
| **Stored in** | Database (extended properties) | Markdown file on disk |
| **Set by** | DBA / database owner | Individual developer |
| **Applies to** | All users of the database | That developer only |
| **Teaches Copilot about** | Business rules, data definitions | Coding style, safety, naming |
| **Persists across machines** | Yes (in the database) | Only if file is synced |
| **Format** | `AGENTS.md` / `CONSTITUTION.md` extended property | `copilot-instructions.md` |

**Closing line:**
> *"Together, these features transform Copilot from a generic AI into a domain expert that knows your data and speaks your team's language — without you having to explain it every time."*
